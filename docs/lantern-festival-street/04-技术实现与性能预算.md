# 04. 技术实现与性能预算

## 1) 技术目标

在不依赖复杂硬件和重型 3D 管线的前提下，实现：
- 无尽花街的 2.5D 透视行进；
- 多角色自动动作循环；
- 滚轮控制前后移动；
- 移动端可运行且不崩溃；
- 可部署到现有静态站点与 GitHub Pages。

## 2) 推荐技术方案

## 2.1 渲染层
- 主方案：`Canvas 2D` 单画布渲染；
- 原因：
  - 单文件可控，方便快速静态页面发布；
  - 对 2.5D 伪透视足够；
  - 可直接手绘民俗动作，无需资产管线。

## 2.2 资源层
- 字体：Google Fonts（标题 + 正文）；
- 图形：尽量用程序绘制，减少大图下载；
- 粒子与烟花使用数学函数生成。

## 2.3 结构层
- `index.html`：结构与文案层
- `style.css`：色板、布局、动效、响应式
- `main.js`：场景生成、投影、动画、输入、循环

## 3) 场景数据结构建议

```js
StreetEntity {
  id,
  lane,          // left | right | center
  type,          // stall | lantern | action
  actionType,    // riddle | dragon | lion | ...
  x,
  z,
  seed,
  phaseOffset,
}
```

运行时：
- `travel`：相机行进距离（滚轮影响）
- `velocity`：行进速度
- `routeLength`：道路总循环长度
- `now`：动画时间基准

通过 `zRelative = (entity.z - travel) mod routeLength` 实现无尽循环。

## 4) 投影与渲染逻辑

## 4.1 2.5D 投影
- 定义地平线 `horizonY`；
- 根据 `zRelative` 计算缩放 `scale`; 
- 屏幕坐标：
  - `screenX = centerX + x * scale`
  - `screenY = horizonY + depthCurve(zRelative)`

## 4.2 遮挡与绘制顺序
- 按 `zRelative` 从远到近排序绘制；
- 先画路与背景，再画摊位，再画人物，再画前景特效；
- 近景对象覆盖远景对象，增强纵深。

## 5) 输入与交互（无主动点击玩法）

- `wheel`：改变速度与方向；
- `touchmove`：移动端上下滑动映射到速度；
- 非交互剧情：所有人物动作自动播放，不要求访问者点击触发。

## 6) 动画实现策略

## 6.1 时间驱动
- 使用 `requestAnimationFrame`；
- 每帧传入 `t`，动作用 `sin/cos` 周期函数实现；
- 每个实体加 `phaseOffset`，避免同频机械运动。

## 6.2 动作函数拆分
- `drawRiddleScene()`
- `drawLanternViewingScene()`
- `drawDragonDanceScene()`
- `drawLionDanceScene()`
- `drawStiltWalkScene()`
- `drawYanggeScene()`
- `drawTangyuanScene()`

保持函数单一职责，便于后续细化。

## 7) 性能预算

目标帧率：
- 桌面端：55-60 FPS
- 中端手机：30-45 FPS

预算建议：
- 屏幕可见实体总数控制在 120-180；
- 每帧绘制路径段避免无限增长；
- 大型阴影和模糊慎用（集中在关键灯光）；
- 设备像素比上限限制为 2，降低超高分屏压力。

## 8) 可访问性与降级

- `prefers-reduced-motion`：
  - 降低粒子、烟花、镜头摆动；
  - 保留核心行进与场景识别。
- 文案层保持固定可见，提供清晰操作提示。
- 小屏自动缩小文字与减少装饰层，避免遮挡。

## 9) 发布与部署

- 将页面放在 `web/lantern-street/`；
- 在 Vite 多入口配置中加入 `lanternStreet`；
- 首页 hub 增加入口卡片；
- GitHub Pages 构建后可通过 `/lantern-street/` 访问。
