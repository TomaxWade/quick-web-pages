# 02. 场景元素与动画导演脚本

## 1) 场景对象总表

为了让街景“极繁但不乱”，对象分为 7 组：

1. 道路与建筑基底
- 中央主街（可反光材质感）
- 两侧店铺轮廓、牌楼、摊棚
- 街道边缘花纹与灯带

2. 光源系统
- 近景灯笼（高亮 + 晃动）
- 中景灯串（连续带状）
- 远景天灯与烟花点

3. 摊位系统
- 糖画摊、花灯摊、元宵摊、字谜摊
- 每个摊位含招牌旗幡、桌案、灯罩

4. 人物群像系统
- 单人、双人、小队列
- 围观者、摊主、表演者

5. 民俗动作系统（指定必须出现）
- 猜灯谜
- 赏花灯
- 舞龙
- 舞狮子
- 踩高跷
- 扭秧歌
- 吃元宵

6. 气氛特效系统
- 光晕脉冲
- 纸屑/花瓣/火星微粒
- 局部薄雾

7. UI 叙事系统
- 主标题
- 引导文案
- 当前里程提示（可选）

## 2) 动画节拍设计

## 2.1 全局时间轴
- 60 秒为一个“导演循环段”；
- 但道路对象按位置循环，不按时间重置，保证“无尽路感”；
- 全局时间用于控制动作同步和节日节奏变化。

## 2.2 动画层级
- L1 背景层：慢速漂移（烟花、云雾）
- L2 建筑层：几乎静态，仅轻微呼吸光
- L3 灯笼层：中速摆动 + 光晕脉冲
- L4 人物层：动作循环（核心叙事）
- L5 前景粒子层：快速掠过增强临场感

## 3) 七类民俗动作导演脚本

### A. 猜灯谜
镜头语义：停步讨论、手指灯谜牌。
动画要点：
- 一人抬手指向灯谜条幅；
- 一人点头或托腮思考；
- 灯谜牌随风摆动，形成轻微随机感。

### B. 赏花灯
镜头语义：抬头仰望、缓慢移动。
动画要点：
- 两名角色头部上抬，身体轻晃；
- 背后大型灯组做柔和明灭；
- 偶发手势变化（指向、合掌称赞）。

### C. 舞龙
镜头语义：高能主秀。
动画要点：
- 龙身由多段节点组成，沿正弦波摆动；
- 龙头振幅略大于龙身；
- 鼓点感通过节拍缩放和位移实现。

### D. 舞狮子
镜头语义：灵动、弹跳。
动画要点：
- 狮头上下点跃，狮身跟随略延迟；
- 偶发“蹲-起-跳”动作三连；
- 周围观众做鼓掌节奏动作。

### E. 踩高跷
镜头语义：高挑、稳中带险。
动画要点：
- 人物腿部比例拉长，步幅更大；
- 躯干保持平衡微摆；
- 行进速度略慢，突出“稳”。

### F. 扭秧歌
镜头语义：队列欢腾。
动画要点：
- 两到三人左右摆胯同步；
- 手部彩扇/彩绸摆动形成拖影；
- 动作频率中等，适合长时间循环。

### G. 吃元宵
镜头语义：生活温度。
动画要点：
- 桌面碗中蒸汽上升；
- 人物端碗、吹气、点头；
- 摊主递碗动作间歇出现。

## 4) 空间编排规则

## 4.1 街段模块化
将无尽街道拆为 8-12 个“街段模板”，每段包含：
- 左右摊位组合
- 1-2 个民俗主动作
- 灯笼密度参数

循环时随机但受约束：
- 相邻段避免同一动作连续出现超过 2 次；
- 每 4 段至少出现一次高能动作（舞龙/舞狮）。

## 4.2 前中后景比例
- 前景：20%（粒子、近灯、牌匾）
- 中景：55%（人物动作与摊位）
- 远景：25%（街道延伸、远灯、烟花）

## 5) 镜头运动与滚轮映射

滚轮行为映射为“步行速度”：
- 向下滚：向前走（街景迎面而来）；
- 向上滚：向后退（已过街景回退）；
- 停止滚动后保持缓慢惯性，再逐渐减速。

额外规则：
- 设定最小前进基速，保证场景不会“完全停死”；
- 对移动端支持上下滑动映射相同逻辑。

## 6) 防重复疲劳机制

- 动作相位偏移：同类角色不同步，避免机械“齐步”；
- 灯笼亮度加入低频噪声；
- 烟花随机在顶部不同区域触发；
- 文案提示每段更新一句节庆短句（可选）。

## 7) 验收标准（动画维度）

- 7 种指定动作全部可见且可识别；
- 任意连续 45 秒至少出现 5 类动作；
- 滚轮前后都能维持透视一致和顺滑；
- CPU 占用可控，不出现明显掉帧卡顿。

本导演脚本确保页面不是“几张素材轮播”，而是一个可持续运转的节庆空间。
